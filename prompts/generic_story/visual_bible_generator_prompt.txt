Return STRICT VALID JSON ONLY.

The visual bible will be used for:

# VISUAL DESIGN RULES

1. Every recurring character must have ONE fixed appearance.

2. Never provide alternatives.

3. Never use uncertain alternatives. Avoid words and phrases such as:

   * maybe
   * possibly
   * could be
   * either
   * or

4. Clothing must have:

   * exact type
   * exact colors
   * exact patterns
   * exact footwear

5. Hair must have:

   * color
   * length
   * style
   * visible structure
   * exact visible count for braids, ponytails, buns, clips, ribbons, parting, and loose strands when present

6. Accessories must specify:

   * always present
   * sometimes present
   * never present

7. If a detail is unknown from the story, invent missing visual details exactly once in this Visual Bible: face type, skin tone, eye shape/color, hair, outfit, footwear, accessories, distinctive features, and body scale. Once invented, these details become hard locks for the cover and all pages.

8. If clothing pattern is not story-important, set pattern to "plain fabric, no print, no embroidery, no motifs, no stripes, no dots".

9. Hair locks must explicitly forbid extra braids, missing braids, duplicate hair tails, swapped sides, and loose braid-like strands when tied hair is used.

10. Accessory locks must explicitly forbid missing fixed accessories and random added accessories.

11. face_lock must include face shape, skin tone, eye shape, and eye color.

12. outfit_lock must include clothing type, exact colors, pattern, and footwear.

13. forbidden_variations must include hairstyle changes, outfit/color changes, footwear changes, missing fixed accessories, random added accessories, skin tone changes, and eye color changes.

# CROSS-STORY VARIETY RULES

Use Visual Diversity Seed as the design seed for this story's cast. Retry for the same workflow must preserve the same designs.

Design each story's cast as visually distinct from generic/default child and parent templates. Do not reuse the same repeated child look across stories.

Do not default every girl to the same pigtails, pink kurta, round face, and brown eyes. Do not default every boy to the same short hair, orange shirt, round face, and shorts. Do not default every parent or grandparent to the same face, clothing, hair, and color palette.

Invented designs must fit the story setting, culture, age group, role, relationship, and activity. For Indian/desi, family, school, classroom, festival, or neighborhood stories, use context-appropriate clothing and details, but vary outfit type, colors, hair structure, footwear, accessories, body scale, and face shape across characters and across different stories.

Within the same story, make each named human character visually distinguishable unless the story explicitly requires matching uniforms, twins, siblings, or a dress code.

Every character must have a design_fingerprint summarizing the locked look. The cast_design_fingerprint must summarize how this story's cast is visually distinct.
