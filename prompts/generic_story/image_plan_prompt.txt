# CONTRACT

Use the Visual Bible as the only source for character appearances, locations, and important objects.

Do not redesign characters, change appearances, add characters, add story events, or change story events.

Use Scene Plan image_brief and visual_direction as source guidance for page staging, shot type, focal subject, lighting, mood, key visual elements, and page-to-page visual continuity.

Do not ignore scene_plan.pages[].image_brief or scene_plan.pages[].visual_direction when present.

Cover:
* Plan a finished front book cover, not page 1, the first interior scene, or any single page composition.
* Base it on the full story summary/arc and Scene Plan cover_brief when available.
* Show the story genre, emotional promise, moral, title, and hero-led focal hierarchy.
* Use overview, symbolic, or hero-plus-world composition when helpful.
* Do not use a seated group/family portrait unless the story is specifically about that portrait.
* Include only useful supporting characters/objects.
* cover.characters must use exact Visual Bible names only.
* If cover.characters is not empty, cover.continuity_notes must include Visual Bible locks for every visible character: face, hair, outfit, fixed accessories, body scale when present, and forbidden variations.
* cover.title_text must exactly match the story title.
* title_layout must reserve clean readable title space. No banners, labels, cards, boxes, UI panels, black rectangles, or overlap from characters/objects/background.
* composition must mention the title area and main story/genre signal.
* book_cover_prompt must be compact, include exact title text/title placement, state front cover based on the whole story, and may reference exact Visual Bible appearance/locked outfit without inventing appearance details.

Pages:
* Create exactly one image-plan page for every Story JSON page, in the same order.
* Each image-plan page.page must equal the corresponding Story JSON page.page_number.
* Output pages.length must equal Story JSON Page Count exactly.
* Output page values must equal Story JSON Page Numbers exactly, in the same order.
* Story JSON is the source of truth for page count. Do not use age-band ranges, Scene Plan page count, or adaptation_strategy.selected_page_count if they differ from Story JSON pages.
* Each page must show the corresponding scene only.
* Each page must show one clear visual moment, not a split scene, montage, collage, sequence, or multi-vignette page.
* If the Scene Plan or Story JSON contains several actions on one page, choose the strongest single visual moment and imply the other actions through pose, expression, object state, or background detail.
* characters is an allow-list for visible characters in that page. Do not include parents, grandparents, siblings, helpers, recurring characters, or background people unless physically visible in that scene.
* characters must use exact Visual Bible names only. Do not use vague labels such as "family", "friends", "children", "villagers", or "classmates" when named characters are available.
* Each character listed in characters must appear exactly once in that page plan. Do not plan duplicate depictions, second heads, reflections, portraits, photos, head-only background figures, partial duplicate bodies, or lookalike versions of the same character.
* If parents/helpers are present but not central, place each one as a single connected background figure or clear upper-body figure with locked face, hair, outfit, footwear, and accessories. Never describe floating heads, detached faces, cropped heads, or partial child/adult cutouts.
* Preserve character, object, and location continuity.
* Preserve the Scene Plan page visual_direction: shot_type, composition, focal_subject, lighting, mood, and key_visual_elements.
* Convert scene_plan.pages[].visual_direction into concrete image-plan camera_shot, composition, environment, visual_focus, and continuity_notes.
* Every visible named character, including side characters, must keep the same Visual Bible identity across all images.
* For every visible character on every page, continuity_notes must include the character's Visual Bible locks: face, hair/fur, skin or body color, outfit, fixed accessories, body scale/age/species, and forbidden variations when present.
* If a character appears on multiple pages, repeat that character's same lock notes consistently. Do not weaken side-character locks on later pages.
* Do not change clothing, colors, hairstyle, face, body shape, age, accessories, or species unless the story explicitly requires a change.
* Apply basic scene etiquette. In temples, prayer rooms, sacred spaces, or homes where shoes are removed, plan characters barefoot, in socks, or with footwear placed neatly near the entrance. On beds, mattresses, sofas used for sleeping, or bedding, do not plan outdoor shoes on feet; place footwear beside the bed/floor if needed. This is a scene-etiquette override, not a character redesign.
* Important named characters should have faces visible and bodies complete enough to identify them. Do not crop off heads, faces, hands, feet, tails, wings, or signature objects unless the scene specifically requires a closeup.
* Avoid adding unnamed background characters. If a crowd is required, keep it simple and never let background figures resemble or replace named characters.
* Show clear emotion, strong focal point, varied framing, child-friendly age-appropriate composition.
* Each page should feel like a premium finished illustration, not a basic spot drawing: crisp focal subject, polished lighting, detailed but uncluttered environment, expressive faces, clean silhouette, and strong depth.
* Avoid repeated poses, nursery/mat setups, object layouts, or character groups unless the scene repeats them.
* Do not use composition, visual_focus, or continuity_notes to request extra partial heads, duplicate faces, head-only cutouts, or duplicate versions of any named character.
* Story pages must not request title text, top text, captions, subtitles, narration text, story prose, floating text, UI text, or overlay text.
* allowed_in_scene_text is the only place to request readable text on story pages. Default it to [].
* Populate allowed_in_scene_text only for short physical object-bound text explicitly needed in the scene, such as "blackboard: HAPPY RAKHI CELEBRATION", "chair labels: Saavi, Myra, Aarav", "classroom sign: Welcome", or "book cover: Neem Care".
* If allowed_in_scene_text is empty, the page must contain no readable text.

Camera shot values: wide, medium, closeup, overhead, low_angle, eye_level.

# FIELD NOTES

book_cover_prompt: compact final cover direction; include title text, title placement, hero/supporting characters, story objects, genre signal, emotion, and clear title area; do not invent appearance details.

visual_focus: single most important illustration moment.

composition: visual layout, for example "Hero in left foreground, treasure map centered, forest path leading into background."
For story pages, composition must describe a single unified illustration. Do not use split scene, panel, panels, vignette, vignettes, montage, sequence, or collage unless the source story explicitly requests that format.

environment: brief setting description.

allowed_in_scene_text: readable text that may appear physically on scene objects only. Use [] unless the page explicitly needs object-bound text such as blackboard writing, chair labels, classroom signs, or book covers. Do not use this for captions, title text, story prose, narration, subtitles, floating text, or top/bottom overlay text.

continuity_notes: character/object/location continuity only; must not add characters or objects outside characters/important_objects.
Include visual_direction continuity when relevant, such as recurring visual element, lighting, mood, composition, or important object continuity.

For page continuity_notes, include one concise lock note per visible named character. Example format:
"Meera lock: same round face, black bob hair, yellow kurta, blue leggings, small silver anklet; no outfit color changes."

For side characters, use the same format and the same strictness as the hero.

13. Side characters must have the same level of continuity detail as the hero.
14. When Scene Plan page visual_direction is present, image-plan camera_shot, composition, visual_focus, environment, and continuity_notes must reflect it.

17. Each listed character appears once only; no duplicate heads, duplicate faces, floating heads, detached faces, head-only background figures, or partial duplicate bodies.
18. Scene etiquette is respected: no outdoor shoes inside temples/prayer rooms/sacred spaces or on beds/bedding; removed footwear may appear beside the entrance or bed.
19. Do not redesign characters or modify Visual Bible appearance details.
20. Story pages must use one unified illustration moment. Do not use split scene, panel, panels, vignette, vignettes, montage, sequence, or collage unless explicitly requested by the source story.
21. Return STRICT VALID JSON ONLY.
